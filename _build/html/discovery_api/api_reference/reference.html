3. EdX Discovery API Reference#

This API reference includes information about the endpoints of the edX Discovery API, including the request, response data, and examples. You can use any technology to send REST API requests as HTTP messages to the edX API. The example API requests shown in this reference use the curl command-line program to show the syntax and data for a request in a way that is easy to read. To view all these endpoints hosted by Discovery service go to /api-docs/

3.1. Taxonomy Endpoints#

The following endpoints are available in Taxonomy application under the Discovery API.

  • /skills

    The skills are extracted using course’s full description by third party tools and services. Skills are linked with courses to help learners choose their courses for their desired skills. You can make GET calls to the /skills endpoint to get all the skills available in the system. See Skills List Endpoint for more details.

  • /skills/{skill_id}

    You can make GET calls to the /skills/{skill_id} endpoint to get details for a specified skill. For details, see skills/{skill_id} Endpoint.

  • /jobs

    We also collect and save the data about jobs that are related to skills. We extract this information using third party tools and services. Jobs are linked with the skills to help learners align their career plans with their course work. You can make GET calls to the /jobs endpoint to get all the jobs. See, Jobs List Endpoint for more details.

  • /jobs/{job_id}

    You can make GET calls to the /jobs/{job_id} endpoint to get details of a specified job. For details, see jobs/{job_id} Endpoint.

  • /jobpostings

    JobPostings contains the number of job postings with posting companies and median salary information. You can make GET calls to the /jobpostings endpoint to get all the job postings available in the system. See JobPostings List Endpoint for more details.

  • /jobpostings/{jobpostings_id}

    You can make GET calls to the /jobpostings/{jobpostings_id} endpoint to get details of a specified JobPostings object. For details, see jobpostings/{jobpostings_id} Endpoint.

3.1.1. Taxonomy Endpoints Response layout#

Responses to GET requests for the edX Discovery API frequently contain the results response value. The results response value is a variable that represents the intended object from the GET request. Here are the details.

Field

Data Type

Description

count

integer

The number of objects in the system.

next

string

The URL for the next page of results.

previous

string

The URL for the previous page of results.

results

array

A list of objects of relative data.

For example in Skills List Endpoint Each top-level object in the results array represents a skill. See skills for information about the fields in a skill item in the results.

3.2. Skills List Endpoint#

GET call to the /skills endpoint returns a list of all the available skills in the system.

3.2.1. Method and Endpoint#

Method

Endpoint

GET

/taxonomy/api/v1/skills/

3.2.2. Example Request#

curl -X GET \
https://discovery.edx.org/taxonomy/api/v1/skills/ \
-H "Authorization: JWT {access token}"

3.2.3. Parameters#

You can use an optional page_size parameter to specify the number of records you want to load in a single page. If not provided the default of 20 is used for page_size. For example:

curl -X GET \
https://discovery.edx.org/taxonomy/api/v1/skills/?page_size=10 \
-H "Authorization: JWT {access token}"

3.3. skills/{skill_id} Endpoint#

GET call to the skills/{skill_id} endpoint returns information about a single skill. In the GET call, The information returned is described in Skill Fields.

3.3.1. Method and Endpoint#

Method

Endpoint

GET

/taxonomy/api/v1/skills/{skill_id}

3.3.2. Example Request#

curl -X GET \
https://discovery.edx.org/taxonomy/api/v1/skills/14 \
-H "Authorization: JWT {access token}"

3.3.3. Response Values#

The GET https://discovery.edx.org/taxonomy/api/v1/skills/{skill_id} request returns the response values described in Skill Fields.

3.3.4. Fields in a Skill#

string

3.4. Jobs List Endpoint#

GET call to /jobs endpoint returns a list of all the available jobs in the system.

3.4.1. Method and Endpoint#

Method

Endpoint

GET

/taxonomy/api/v1/jobs/

3.4.2. Example Request#

curl -X GET \
https://discovery.edx.org/taxonomy/api/v1/jobs/ \
-H "Authorization: JWT {access token}"

3.4.3. Parameters#

You can use an optional page_size parameter to specify the number of records you want to load in a single page. If not provided the default of 20 is used for page_size. For example:

curl -X GET \
https://discovery.edx.org/taxonomy/api/v1/jobs/?page_size=10 \
-H "Authorization: JWT {access token}"

3.6. JobPostings List Endpoint#

GET call to the /jobpostings endpoint returns a list of all the available JobPostings in the system.

3.6.1. Method and Endpoint#

Method

Endpoint

GET

/taxonomy/api/v1/jobpostings/

3.6.2. Example Request#

curl -X GET \
https://discovery.edx.org/taxonomy/api/v1/jobpostings/ \
-H "Authorization: JWT {access token}"

3.6.3. Parameters#

You can use an optional page_size parameter to specify the number of records you want to load in a single page. If not provided the default of 20 is used for page_size. For example:

curl -X GET \
https://discovery.edx.org/taxonomy/api/v1/jobpostings/?page_size=10 \
-H "Authorization: JWT {access token}"

3.7. jobpostings/{jobpostings_id} Endpoint#

3.7.1. Method and Endpoint#

/taxonomy/api/v1/jobpostings/{jobpostings_id}

3.7.2. Example Request#

-H "Authorization: JWT {access token}"

3.7.3. Response Values#
